# 🔴 项目 E：Vibe Code 一个实用工具

> **难度**: 综合 ｜ **耗时**: 2-3 小时 ｜ **工具**: Google AI Studio  
> **对应课程**: 07 App Building — 全部

## 为什么做这个项目

Vibe Coding = 用自然语言描述需求，让 AI 帮你写代码并构建完整应用。不需要编程基础，不需要看懂代码，你只需要清楚地描述"我要什么"。

这个项目带你从零构建一个**真正能用的 Web 工具**。

---

## 选择你要做的工具

选一个你会真正用到的（推荐 A 或 B）：

| 方案 | 做什么 | 实用场景 | 难度 |
|------|--------|----------|------|
| **A. 会议摘要器** | 粘贴会议纪录 → 自动生成摘要 + 待办 | 每天都能用 | ⭐⭐ |
| **B. 决策分析器** | 输入选项和条件 → 给出分析建议 | 面临重要选择时 | ⭐⭐ |
| **C. 学习卡片生成器** | 输入知识点 → 生成问答闪卡 | 备考/学习新知识 | ⭐⭐⭐ |
| **D. 周报生成器** | 输入本周工作 → 生成结构化周报 | 每周写周报时 | ⭐⭐ |

以下用 **方案 A（会议摘要器）** 作为教学示例，其他方案只需修改需求描述即可。

## Part 1：明确需求（15 min）

做应用之前，先想清楚要什么。在 Gemini 中规划：

```
我想做一个"会议摘要生成器" Web 应用，帮我细化需求：

基本功能：
- 用户粘贴会议记录（文字）
- AI 自动生成：会议摘要、关键决策、待办事项列表
- 可以复制结果或导出

请帮我：
1. 补充我没想到的实用功能（最多 3 个）
2. 设计用户操作流程（几步完成？）
3. 定义输出格式（每个模块包含什么）
4. 给出 UI 布局建议（简洁优先）
```

---

## Part 2：在 AI Studio 中构建（60 min）

### 第 1 步 — 打开 Google AI Studio

1. 访问 [Google AI Studio](https://aistudio.google.com/)
2. 选择 "Build with Gemini" 或创建新项目

### 第 2 步 — 描述你的应用

在 AI Studio 中使用以下 Prompt 开始构建：

```
请帮我创建一个"会议摘要生成器" Web 应用。

功能需求：
1. 一个大文本框，用于粘贴会议记录
2. 一个"生成摘要"按钮
3. 点击后，AI 分析会议内容并输出：
   - 📋 会议摘要（100 字以内）
   - ✅ 关键决策（编号列表）
   - 📌 待办事项（包含负责人和截止日期，如果提到的话）
   - ⚠️ 未解决问题（如果有）
4. 每个输出模块有独立的"复制"按钮
5. 支持重新生成

UI 要求：
- 简洁现代的界面，浅色主题
- 顶部标题 + 说明文字
- 左侧输入区，右侧输出区（或上下布局）
- 移动端友好
- 有加载状态提示
```

### 第 3 步 — 迭代改进

第一版出来后不会完美，用以下方式迭代：

**改进 UI**
```
请优化界面：
1. 按钮加上图标
2. 输出区域用卡片样式，每个模块一张卡片
3. 加上一个示例按钮，点击后自动填充示例会议记录
4. 空状态时显示使用说明
```

**改进 AI 输出质量**
```
请优化 AI 的 System Prompt，确保：
1. 摘要抓住最关键的信息，不要流水账
2. 待办事项格式统一为：[ ] 事项内容 — @负责人 — 截止日期
3. 如果会议记录比较乱/不完整，给出友好提示而不是硬编
4. 支持中英文会议记录
```

**加新功能**
```
请添加以下功能：
1. 语气选择：正式版 / 轻松版 摘要
2. 长度控制：简短 / 标准 / 详细
3. 历史记录：保存最近 5 次生成结果（本地存储）
```

## Part 3：测试和完善（30 min）

### 第 1 步 — 准备测试数据

```
请帮我生成 3 段模拟会议记录用于测试：

1. 产品评审会（10 人参与，讨论新功能优先级）
2. 项目周会（5 人参与，简短状态更新）
3. 头脑风暴会（7 人参与，讨论混乱无明确结论）

每段 200-300 字，包含人名、时间、具体讨论内容。
第 3 段故意写得比较乱，用来测试 AI 的处理能力。
```

### 第 2 步 — 边界测试

用以下场景测试应用：

| 测试场景 | 预期行为 |
|----------|----------|
| 粘贴正常会议记录 | 正确生成摘要和待办 |
| 粘贴很短的文本（1-2 句话） | 友好提示"内容太少" |
| 粘贴非会议内容（新闻、代码等） | 识别并提示"似乎不是会议记录" |
| 粘贴英文会议记录 | 正常处理，输出中文 |
| 什么都不输入就点按钮 | 提示"请先输入内容" |

### 第 3 步 — 修复问题

```
在测试中发现以下问题，请修复：
[列出你发现的问题]

优先级：
1. 功能性 Bug（必须修复）
2. 体验问题（应该修复）
3. 美化建议（有余力再做）
```

### 部署应用

AI Studio 构建的应用通常可以直接生成分享链接：

1. 检查应用是否正常运行
2. 生成分享链接
3. 在手机上测试一下

### 记录开发过程

```
请帮我生成一份开发日志，包含：

1. 应用名称和功能简介
2. 使用的技术栈（AI Studio 自动选择的）
3. 开发过程（我给了哪些关键 Prompt）
4. 迭代了几次，每次改了什么
5. 测试中发现的问题和解决方法
6. 总耗时
7. 心得体会（Vibe Coding 的优势和局限）
```

- AI 构建应用的能力边界在哪里
