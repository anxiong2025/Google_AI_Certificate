# 🟡 项目 B：个人知识库 + 研究助手

> **难度**: 进阶 ｜ **耗时**: 2-3 小时 ｜ **工具**: NotebookLM / Gemini Deep Research / Gemini Gems  
> **对应课程**: 02 Brainstorming & Planning — Module 03, 03 Research & Insights — 全部

## 为什么做这个项目

信息过载是现代人的核心痛点。这个项目帮你搭建一套 **"收集 → 整理 → 提问 → 研究 → 咨询"** 的完整知识工作流，做完就有一个能持续使用的个人研究系统。

---

## Part 1：用 NotebookLM 搭建知识库（45 min）

### 第 1 步 — 选定领域，收集资料

选一个你真正关心的话题。以下是几个推荐领域：

| 领域 | 适合谁 | 资料来源建议 |
|------|--------|-------------|
| AI 行业趋势 | 技术从业者 | AI 报告、技术博客、论文摘要 |
| 个人理财/投资 | 所有人 | 理财书摘、投资分析文章 |
| 健康管理 | 关注健康的人 | 营养学文章、运动指南 |
| 某个专业技能 | 想深耕的人 | 教程、最佳实践文档 |

**收集 8-15 份资料**，格式可以是：
- PDF 文档（报告、电子书章节）
- Google Docs
- 网页链接
- YouTube 视频链接
- 复制粘贴的文本

### 第 2 步 — 创建 NotebookLM 知识库

1. 打开 [NotebookLM](https://notebooklm.google.com/)
2. 点击 "New Notebook"
3. 上传/添加你收集的所有资料

### 第 3 步 — 测试知识库问答

把以下 Prompt 模板复制到 NotebookLM 中测试（替换 `[你的领域]`）：

**问答测试**
```
关于 [你的领域]，请回答以下问题并标注信息来源：
1. 目前最重要的 3 个趋势是什么？
2. 初学者最容易犯的错误有哪些？
3. 你推荐的入门路径是什么？
```

**摘要生成**
```
请对所有上传的资料做一个综合摘要，要求：
- 按主题分类（不是按文档分类）
- 标注每个观点来自哪份资料
- 指出不同资料之间的矛盾观点（如果有的话）
- 最后列出 3 个值得深入研究的问题
```

**知识图谱**
```
请梳理这些资料中涉及的核心概念，并用以下格式呈现它们的关系：

概念 A → 影响 → 概念 B
概念 C ← 依赖 ← 概念 D
概念 E ≈ 类似 ≈ 概念 F

最后用一段话描述这些概念之间的整体关系。
```

### 第 4 步 — 生成音频概述

NotebookLM 能将知识库转化为播客式音频，方便通勤时收听：

1. 点击 "Audio Overview"
2. 等待生成（通常 2-5 分钟）
3. 下载收听，评估质量

## Part 2：用 Deep Research 做专题研究（30 min）

### 第 1 步 — 发起深度研究

打开 [Gemini](https://gemini.google.com/)，切换到 Deep Research 模式,使用以下模板：

```
请对以下主题做深度研究：

主题：[基于 Part 1 中发现的值得深入研究的问题]

研究要求：
- 覆盖最近 12 个月的信息
- 关注来自权威来源的数据和观点
- 呈现不同立场的观点对比
- 最后给出你的综合判断

输出格式：
1. 研究背景（100 字）
2. 关键发现（分点列出，每点附来源）
3. 不同观点对比（表格）
4. 数据和趋势（如有）
5. 结论和建议
```

### 第 2 步 — 交叉验证

```
请将这份 Deep Research 的研究结果与我 NotebookLM 知识库中的信息做对比：

知识库中的关键观点：
[从 Part 1 的问答结果中粘贴 3-5 个关键观点]

请分析：
1. 哪些观点得到了 Deep Research 的验证？
2. 哪些观点被新信息更新或推翻了？
3. Deep Research 发现了哪些知识库中没有的重要信息？
```

## Part 3：用 Gems 创建专家顾问（30 min）

### 第 1 步 — 创建你的专属 Gem

在 Gemini 中，进入 Gems 功能，使用以下配置模板：

**Gem 名称**: `[你的领域] 专家顾问`

**Gem 指令**（复制进去）：
```
你是一位 [领域] 领域的资深专家，具备以下背景：

专业背景：
- 在 [领域] 有 15 年以上从业经验
- 熟悉最新的行业趋势和最佳实践
- 擅长将复杂概念用简单语言解释

沟通风格：
- 回答问题时先给结论，再给论据
- 主动指出提问者可能忽略的盲点
- 遇到不确定的问题，诚实说明并给出推荐的信息来源
- 用具体案例支撑观点

工作方式：
- 如果问题太宽泛，先帮我缩小范围
- 每次回答后，主动建议 2-3 个值得追问的方向
- 区分"已验证的事实"和"个人判断"
```

### 第 2 步 — 模拟咨询对话

用以下问题测试你的 Gem：

```
我想在 [领域] 方面做出改进/学习/决策。

我的现状：[简述你的情况]
我的目标：[你想达到什么]
我的困惑：[你不确定的地方]

请像一个专业顾问一样，帮我分析现状并给出行动建议。
```

```
以下是我从研究中得到的结论：
[粘贴 Part 2 的研究结论]

请从专家角度评估：
1. 这些结论中哪些是靠谱的？
2. 哪些可能过于乐观或简化？
3. 如果我基于这些结论做决策，需要注意什么风险？
```

## 最终产出检查清单

完成这个项目，你应该拥有：

- [ ] 一个包含 8-15 份资料的 NotebookLM 知识库
- [ ] 知识库的综合摘要和概念关系图
- [ ] 一份 Audio Overview（音频概述）
- [ ] 一份 Deep Research 研究报告
- [ ] 交叉验证分析（知识库 vs 最新研究）
- [ ] 一个配置好的专属领域 Gem
- [ ] 至少一轮专家咨询对话记录

## 持续使用建议

这不是做完就扔的项目——它是你的 **个人研究系统**：

- **每周** 往 NotebookLM 添加 2-3 份新资料
- **每月** 用 Deep Research 做一次趋势更新
- **遇到决策** 先问 Gem 做预判，再做人工验证
